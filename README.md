# 奇趣财富岛

原创卡通风网页端多人联网大富翁游戏。本项目玩法机制参考了传统大富翁游戏，也受到 4399 平台《星猫大富翁》的启发。

本项目为非官方原创作品，与 4399 及相关权利方无隶属、授权或合作关系；除说明灵感来源外，不使用原作角色、美术、地图、音乐、代码或其他素材。

## 技术栈

- 前端：React + TypeScript + Vite，SVG 棋盘与卡通 UI。
- 后端：Node.js + TypeScript + Express + Socket.IO。
- 同步：后端是唯一权威状态来源，客户端只负责展示和发送操作请求。
- 存储：MVP 使用内存房间状态，后续可替换 Redis / SQLite。
- 联机：支持 2 到 8 人、房间号加入、准备、房主开始、房主踢人、断线重连。
- AI 补位：房主可在大厅添加 / 移除服务端 AI 玩家，AI 开局后自动行动。
- 安全重连：浏览器在本机保存私有重连令牌，服务端只保存哈希；公开的玩家 ID 不能用于冒充其他玩家。

## 启动

```bash
cd monopoly-online
npm install
npm run dev
```

默认地址：

- 前端：http://localhost:5173
- 后端：http://localhost:4000
- 健康检查：http://localhost:4000/health

也可以分别启动：

```bash
npm run dev:server
npm run dev:client
```

如果 `4000` 端口被占用，先关闭旧服务端进程，或用 `PORT` 指定新端口。

## 当前玩法

- 80 格地图：68 个普通路线节点 + 6 格监狱支路 + 6 格医院支路。
- 8 个可选出生点：内圈四角与四边中点；支持统一或独立出生。
- 房主可以用 AI 补位凑满 2 到 8 人；AI 使用服务端原有规则函数自动掷骰、购买、升级、传送、借款、抵押和结束回合。
- 地产可购买、升级到 4 级、收租、抵押、赎回，并按 9 个地产套装提供租金翻倍。
- 银行支持存款、取款、信用借款、还款、月结利息、彩券兑换和提前离开监狱 / 医院。
- 股票系统包含 10 只原创虚拟股票、交易日、周末休市、日终统一结算、手续费、持仓成本和个人情报。
- 当天买入股票不参与当天收盘涨跌，新买入批次当天没有收益或亏损。
- 彩票支持一次多张购买、逐张开奖和中奖明细。
- 技能卡、好运、厄运、游乐场、传送门、税收、随机事件均由服务端结算并同步。
- 游戏结束排行榜可关闭，关闭后不遮挡地图，可再次查看。
- 游戏结束后可重新开局。

完整规则、彩票概率、股票买卖规则、银行规则和结算说明见 [GAME_RULES.md](./GAME_RULES.md)，全部技能卡见 [SKILL_CARDS.md](./SKILL_CARDS.md)，地图坐标和路线规划见 [MAP_PLAN.md](./MAP_PLAN.md)。

AI 补位的服务端封装、决策范围和后续扩展计划见 [AI_FILL_DESIGN.md](./AI_FILL_DESIGN.md)。

## 多人本地测试

1. 第一个浏览器窗口输入昵称并创建房间。
2. 第二个浏览器窗口或无痕窗口输入昵称和房间号加入。
3. 两名玩家都点击准备。
4. 房主调整规则后开始游戏。
5. 玩家轮流掷骰，移动后处理地块事件。
6. 刷新页面时，客户端会使用本地保存的 `roomId + playerId` 自动重连。

## 管理员模式测试

如果你不想为了测试某个技能、某张事件卡、某种地产状态或某段股票流程，从开局一路慢慢玩到那个节点，那么推荐使用现在内置的管理员模式。

管理员模式的定位是：

- 用真实对局界面快速构造测试场景
- 保留接近正常游玩的操作体验
- 大幅缩短手测路径

它不是自动化断言系统，但非常适合日常开发时快速验证玩法是否正常。

### 什么时候可以看到管理员模式入口

管理员模式不是首页按钮，也不是房间页按钮。

只有满足下面条件时，入口才会出现：

1. 你是当前房间的房主
2. 游戏已经开始，进入了正式游戏界面

满足后，房主会在游戏界面右下侧看到 `管理员测试` 按钮。

如果你看不到这个按钮，优先检查：

- 你是不是房主
- 游戏是不是还停留在准备房间，没有开始
- 当前打开的客户端是不是房主对应的那个窗口

### 如何打开管理员模式

建议按下面顺序操作：

1. 启动项目
2. 创建一个房间
3. 再开一个浏览器窗口或无痕窗口，加入第二名玩家
4. 两名玩家都点击准备
5. 由房主开始游戏
6. 进入游戏后，房主点击 `管理员测试`

之所以需要至少两名玩家，是因为当前游戏正常开始仍然需要满足基础开局条件；管理员模式是在“已开局的真实对局”上做状态构造，而不是跳过开局规则直接单机进地图。

### 打开后怎么用

管理员面板打开后，建议先做一件事：

1. 先选中你要测试的目标玩家

之后再根据测试目标，去对应的区域操作。

面板目前分成下面几类能力：

#### 1. 玩家资源

适合快速构造资金和银行相关场景。

你可以直接修改：

- 现金
- 彩券
- 存款
- 贷款本金
- 未付利息

常见用途：

- 测试“钱不够时”的购买或缴费逻辑
- 测试银行借款、还款、月结利息
- 测试彩券兑换、彩票购买前后的状态

#### 2. 技能卡发放

适合快速测试某张技能卡。

你可以直接给指定玩家发放任意技能卡，不需要靠随机抽卡或技能商店慢慢等。

常见用途：

- 测试某张技能卡能否正常使用
- 测试技能卡对玩家、地产、股票或移动的影响
- 测试新增技能卡是否已经正确接入现有系统

#### 3. 事件卡触发

适合快速测试事件类玩法。

你可以直接触发任意事件卡，包括：

- 好运
- 厄运
- 彩票
- 游乐

注意这里是“直接触发效果”，不是把事件卡放进玩家手牌。

常见用途：

- 测试新增好运/厄运事件
- 测试彩票或游乐类随机效果
- 测试事件引发的移动、金钱变化、技能卡变化、股票变化

#### 4. 传送到格子

适合快速测试指定格子。

你可以把任意玩家直接传送到任意格子，并且可以选择：

- 只移动到该格子
- 移动后立即结算该格子的效果

这是测试地图玩法时最常用的能力之一。

常见用途：

- 测试某个地产格的购买、升级、收租
- 测试银行、股票、彩票、游乐场、传送门、税收等格子
- 测试新增格子或修改后的格子逻辑

#### 5. 地产状态

适合快速构造地产局面。

你可以直接设置：

- 这块地产归谁
- 这块地产现在是几级
- 这块地产是否处于抵押状态

常见用途：

- 直接构造“高等级地产收租”场景
- 测试抵押、赎回、租金变化
- 测试地产套装、地产归属切换后的表现

#### 6. 玩家持股

适合快速构造股票账户场景。

你可以直接指定某个玩家拥有哪些股票、拥有多少股。

常见用途：

- 测试持仓显示
- 测试收益率、浮盈浮亏、总资产变化
- 测试某个新增股票是否正常出现在持仓与市场中

#### 7. 股票价格

适合直接验证股票涨跌相关表现。

你可以直接指定某只股票的新价格。

常见用途：

- 测试价格上涨后的收益显示
- 测试价格下跌后的亏损显示
- 测试收盘结算、玩家资产变化、行情显示

#### 8. 拘留状态

适合测试监狱、医院等停留状态。

你可以直接把玩家设置为：

- 无拘留
- 监狱
- 医院

并可直接指定持续回合数。

常见用途：

- 测试跳过回合逻辑
- 测试提前离开监狱或医院
- 测试相关技能卡或银行操作与拘留状态的联动

#### 9. 回合与阶段

适合快速推进或切换游戏节奏。

你可以直接设置：

- 当前行动玩家
- 当前阶段
- 当前回合数
- 已完成行动数
- 是否清空待处理动作

常见用途：

- 测试某个操作只能在特定阶段出现时的表现
- 测试待处理动作是否能被正确清理
- 测试某些“只在当前玩家回合生效”的逻辑

### 推荐测试套路

如果你只是想高效验证某类玩法，推荐直接按下面思路组合使用管理员面板。

#### 测某块地产是否正常

1. 用“地产状态”把目标地产分配给玩家 A，并设置到需要的等级
2. 用“传送到格子”把玩家 B 送到这块地产
3. 勾选立即结算格子效果
4. 观察租金、日志、资产和状态变化是否正确

#### 测某张技能卡是否正常

1. 用“技能卡发放”把目标技能卡发给指定玩家
2. 如果技能需要特定目标，再用“传送到格子”或“地产状态”先构造前置场景
3. 回到正常操作栏中使用该技能卡
4. 观察实际效果、日志和状态变化

#### 测股票涨跌是否正常

1. 用“玩家持股”先给目标玩家一笔持仓
2. 用“股票价格”直接拉升或打压对应股票
3. 查看持仓收益、总资产、市场显示是否同步变化

#### 测监狱或医院流程

1. 用“拘留状态”把目标玩家直接送进监狱或医院
2. 用“回合与阶段”快速切换到目标玩家回合
3. 测试跳过回合、提前离开、相关卡牌或银行操作

### 使用管理员模式时的注意事项

为了避免误解，建议记住下面几点：

- 管理员模式只有房主能用，其他玩家看不到入口
- 管理员操作会直接修改真实对局状态，并同步给房间内所有玩家
- 有些操作会立刻触发正常的服务端结算，不只是改一个前端显示
- 如果你要反复试验，建议专门开一个测试房间
- 如果当前要测的是全新玩法类型，而不是已有类型的新内容，后续可能还需要补新的管理员入口

## 音频素材

公开仓库不包含 MP3、歌词或第三方音效文件。项目会在本地启动时自动扫描：

- `client/public/audio/bgm`
- `client/public/audio/sfx`

把已授权、原创或允许再分发的音乐和音效放入对应目录后，启动 Vite 会自动生成本地的 `client/public/audio/manifest.json`，前端音频面板会显示可选 BGM。没有外部音频时，游戏会使用内置的合成轻音乐和音效。

上述两个目录中的媒体文件、歌词文件和自动生成的清单均被 Git 忽略，因此本机曲库不会被意外提交到公开仓库。请只使用你有权播放和分发的素材；Git LFS 只能解决大文件存储问题，不能提供版权授权。

## 当前限制

- 房间仍是内存状态，服务端重启会丢失。
- 彩票和游乐场仍是随机结算面板，还不是完整小游戏。
- 股票没有 K 线、订单簿、涨跌停和复杂撮合。
- 建议继续补规则引擎单元测试，覆盖移动、分岔、银行、股票、地产、技能卡和破产边界。
